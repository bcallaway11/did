## Background: Two Distinct Moment-Set Designs

The `edid()` implementation and the author's reference code (`benchmark/edid_sim_original.R`)
implement the same estimator *class* but with different moment sets. They are NOT expected to
produce identical ATT or SE values. This is documented in `benchmark/compare_author_vs_edid.R`.

| Property | Author reference | R `edid()` (PT-All) |
|----------|-----------------|---------------------|
| Comparison cohorts (`gp`) | Treated cohorts only | Treated cohorts only |
| Never-treated as `gp` | No | No |
| Period 1 as `tpre` for self-pair (`gp==g`) | Yes (author includes it) | Yes (fixed — included) |
| Period 1 as `tpre` for cross-pair (`gp!=g`) | No | No |
| `gp=Inf` in any PT-All pair | No | No (fixed — removed) |
| Omega estimation | `cov()` of per-pair IFs (n-1 divisor) | Analytical closed form (`cov_nn_edid()`, n divisor) |
| Pairs per cell, benchmark panel (cohorts {3,5,7}, periods 0–9) | 13 | 13 (same) |

**Ground truth for integration tests**: Use the Python `diffdiff` package outputs in
`benchmark/data/py_edid_all_gt.csv`, NOT the author's R reference, because the Python outputs
were generated by the same estimator design as the final `edid()` implementation.

## 1. Behavioral Contract

The following behaviors MUST hold after the fix:

1. Under `pt_assumption = "all"`, `enumerate_valid_pairs_edid()` MUST NOT return any row with `gp = Inf` (or `gp = never_treated_val`).
2. Under `pt_assumption = "all"`, for `gp == target_g`: rows with `tpre == period_1` MUST be present when `period_1 < target_g - anticipation` (self-pair at first period is valid).
3. Under `pt_assumption = "all"`, for `gp != target_g`: rows with `tpre == period_1` MUST NOT be present.
4. Under `pt_assumption = "post"`, the function MUST return exactly the same output as before (exactly one row with `gp = Inf` and `tpre = target_g - 1 - anticipation`, or 0 rows).
5. The Omega matrix for a (g, t) cell under PT-All MUST be symmetric and positive semidefinite.
6. The efficient weights under PT-All MUST sum to 1.0 (within machine epsilon).
7. The EIF vector for any (g, t) cell under PT-All MUST have mean zero within `sqrt(.Machine$double.eps) * 10`.
8. The ATT estimate for the benchmark panel (`benchmark/data/diffdiff_panel.csv`) under PT-All MUST match the Python `diffdiff` reference (`benchmark/data/py_edid_all_gt.csv`) within tolerance 1e-10 for each post-treatment cell.
9. The SE estimate MUST match the Python reference within a tolerance that accounts for the `cov()` vs `cov_nn_edid()` divisor difference: for n units per cohort the SE ratio is `sqrt(n/(n-1))`. For the benchmark panel (90 units in G=0, 60 in G=3, 75 in G=5, 75 in G=7) the maximum expected SE inflation relative to Python is under 1.5%, so the tolerance is `max(se_py * 0.02, 1e-4)` per cell.
10. For the benchmark panel with cohorts {0,3,5,7} and periods 0–9: every post-treatment (g, t) cell must have exactly 13 valid pairs, and no pair may have `gp = Inf`.

## 2. Covariance Divisor Note (SE Tolerance Rationale)

The author's R reference uses `cov(IF_mat)` which divides by `n-1` (Bessel's correction).
Our `cov_nn_edid()` divides by `n`. For a cohort of size `n_c`, the Omega entries are scaled by
`n_c / (n_c - 1)`. Because the SE involves `sqrt(1 / sum(Omega_inv))` and `Omega_inv` scales
inversely, the net effect on the SE is:

    SE_ours / SE_author ≈ sqrt((n_c - 1) / n_c)  <  1

For the benchmark panel: G=3 has 60 units, so the ratio is `sqrt(59/60) ≈ 0.9916` — our SE
would be about 0.84% LOWER than the author's. For the Python reference the divisor behavior
must be confirmed from the benchmark outputs; the tolerance `se_py * 0.02` is conservative.

## 4. Integration Tests Against Author Reference

### Critical Note: Moment Set Mismatch

The author's R reference code (`benchmark/edid_sim_original.R`) and our `edid()` use the SAME
moment set after the fix:
- Both use only treated cohorts as comparison cohorts (no `gp=Inf`)
- Author includes `(gp=g, tpre=period_1)` pairs — so does `edid()` after the fix
- Both exclude `(gp≠g, tpre=period_1)` pairs

However, they differ in how Omega is estimated:
- Author: `cov(IF_mat)` — sample covariance with n-1 divisor
- `edid()`: `cov_nn_edid()` — analytical formula with n divisor

And the author's reference code is written in R while the external ground truth used for numerical
validation is the Python `diffdiff` package output in `benchmark/data/py_edid_all_gt.csv`.

**For ATT point estimates**: use the Python outputs as the exact ground truth (tolerance 1e-10),
because ATT does not depend on the covariance estimator.
**For SE estimates**: the Python outputs use a specific divisor; use a 2% relative tolerance.

### Scenario E3 — Degenerate Self-Pair: Omega Entry for (gp=target_g, tpre=period_1)

This is a structural test, not a runtime test. It verifies the mathematical property:

When `gp == target_g` and `tpre == period_1`, the comparison-cohort vector
`delta_gp = Y_g(period_1) - Y_g(period_1)` is the zero vector. Therefore:
- Term C_j = `cov_nn_edid(delta_g_t_1, 0) == 0`
- Term D (when j == k == degenerate_idx): `cov_nn_edid(0, 0) == 0`
- Omega[degenerate_idx, degenerate_idx] = Term_A + Term_B only

**Verification**: For a panel with g=3, target cell (g=3, t=4), period_1=1:

```r
pairs_with_degen <- enumerate_valid_pairs_edid(3L, c(3L), 1:5, 1L, "all")
# pairs: (gp=3, tpre=1) and (gp=3, tpre=2)
degen_idx <- which(pairs_with_degen$gp == 3L & pairs_with_degen$tpre == 1L)
omega <- compute_omega_star_nocov_edid(3L, 4L, pairs_with_degen, panel_obj, "all")
# Term C and D for degenerate row/col should be 0
# Verify: omega[degen_idx, j] for j != degen_idx equals omega_without_degen_term
# Concrete check: all entries in row/col degen_idx are finite
expect_true(all(is.finite(omega[degen_idx, ])))
expect_true(all(is.finite(omega[, degen_idx])))
expect_true(omega[degen_idx, degen_idx] >= 0)
```

No divide-by-zero or NaN should occur.

### Scenario E4 — Panel With Only Pre-Treatment Periods (No Post-Treatment Cells)

For a panel where all `t < g` (no post-treatment observations for a given group), `edid()` should
return NA ATT for those cells without error. This tests the `empty pairs → NA fallback` path in
`edid-fit.R`.

## 7. Property-Based Invariants

| Property | Verification |
|----------|-------------|
| No `gp=Inf` in PT-All pairs | `all(is.finite(pairs$gp))` for ALL PT-All cells |
| Self-pair includes period_1 | `any(pairs$gp == target_g & pairs$tpre == period_1)` when `period_1 < target_g` |
| Cross-pair excludes period_1 | `!any(pairs$gp != target_g & pairs$tpre == period_1)` |
| Omega symmetry | `max(abs(omega - t(omega))) < 1e-12` |
| Omega PSD | All eigenvalues >= -1e-10 |
| Omega all finite | `all(is.finite(omega))` |
| Weights sum to 1 | `abs(sum(w) - 1) < 1e-12` |
| EIF mean zero | `abs(mean(eif)) < sqrt(.Machine$double.eps) * 10` |
| PT-Post gp=Inf | `all(result$gp == Inf)` for PT-Post |
| PT-Post exactly 1 pair | `nrow(result) == 1L` when `tpre_val != period_1 && tpre_val %in% time_periods` |
